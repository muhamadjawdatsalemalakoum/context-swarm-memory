# Evidence and Reproducibility

This document maps the public CSM claims to the artifacts that back them. It is
written for skeptical readers: every headline number should have a file, a
command, and a limitation attached to it.

## External bar

CSM aims for the normal artifact standard used in computational research:

- ACM artifact review: artifacts should be documented, consistent, complete,
  exercisable, and include evidence of verification and validation.
  <https://www.acm.org/publications/policies/artifact-review-and-badging-current>
- NeurIPS-style reproducibility: experiments should include the code, data, and
  instructions needed to reproduce the main results.
  <https://nips.cc/public/guides/PaperChecklist>
- Practical ML reproducibility: code, data, models/config, dependency versions,
  and a runnable analysis path are more persuasive than screenshots.

These small run artifacts are committed because they are the rows behind the
public charts and tables. Large local caches, embeddings, sidecar indexes, and
ad-hoc experiments remain ignored.

The SOTA bar is deliberately broader than the current saved rows. The current
target ladder and go/no-go rules are in `docs/SOTA_BENCHMARK_PLAN.md`; until
those runs exist, the public SOTA claim is limited to the saved LightRAG
head-to-head plus documented Mem0/HippoRAG blockers.

The verifier hashes the result/config/report files and recomputes the headline metrics
from `results.jsonl`:

```bash
npm run verify:published
```

## Honest limits

- The public benchmark is still single-trial R&D evidence. CSM has measured
  run-to-run variation even at temperature 0, so the strongest claim is the
  observed scaling pattern and head-to-head behavior on these saved rows, not
  universal dominance.
- Latency is hardware-specific. The published latency numbers came from one
  RTX 4090-class local machine and should be treated as operational evidence,
  not a portable benchmark constant.
- Mem0 and HippoRAG are documented as blocked locally, not beaten. The repo
  should not claim a numerical win against systems that did not produce a clean
  local 30-query run.
- Full scientific-grade polish still needs the external DOI to be minted and an
  independent replication report from someone outside the author. The repo now
  includes `CITATION.cff`, `.zenodo.json`, sidecar Docker/lock artifacts,
  `docs/SCIENTIFIC_RELEASE.md`, `docs/REPLICATION_KIT.md`, and
  `npm run bench:trials`. The Gemini 3.5 Flash 160K run is now committed as
  single-trial cross-model evidence, but 3-trial confirmation rows are not yet
  public evidence until a full confirmation run is completed and archived.
